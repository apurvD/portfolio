A full-stack web application to digitize vendor operations, booth scheduling, and product discovery at flea markets.

Overview

Digital Flea Market is a database-driven marketplace platform designed to bring traditional flea markets online. The application enables vendors to manage inventory, schedule booth slots, and showcase products, while giving customers a way to browse merchandise and plan their visits more effectively.

Problem Addressed

Traditional flea markets often lack digital infrastructure for individual vendors. This makes it difficult for customers to discover merchandise in advance, plan their visits, and compare offerings. Vendors also struggle to track inventory and sales performance in a structured way. This project addresses these gaps by providing a centralized, vendor-friendly digital marketplace.

Key Features

  • Vendor dashboard: Manage inventory, booth schedules, and product listings from a single interface.
  • Product CRUD operations: Create, read, update, and delete products with detailed views and categorization.
  • Booth scheduling: Assign and manage booth slots for vendors across different dates or events.
  • Analytics: Aggregate vendor performance metrics such as product counts, inventory status, and activity.
  • Customer-facing views: Browse products and vendors to plan visits and discover merchandise ahead of time.

Technologies & Architecture

Development

  • IDE: Visual Studio Code
  • Source Control: GitLab
  • API Testing: Postman

Database Layer

  • MySQL 8.0: Relational DBMS with ACID compliance using the InnoDB storage engine.
  • Docker Compose: Containerized database deployment for consistent local environments.
  • Key capabilities: Foreign key constraints, transaction support, and normalized schema design.

Backend Layer

  • Node.js & Express: RESTful API server for vendor, product, and analytics endpoints.
  • MySQL2 driver: Database connectivity with parameterized queries for secure data access.
  • API endpoints: Vendor management, product CRUD operations, and analytics aggregation.

Frontend Layer

  • React 18: Component-based UI for dashboards and product views.
  • Tailwind CSS: Utility-first styling for responsive, consistent layouts.
  • React Router: Client-side routing for navigation between vendor dashboards, product pages, and other views.

Implementation Highlights

  • Containerized setup: Application and database run in Docker containers for reproducible environments.
  • Data integrity: Use of foreign keys and transactions to maintain consistent relationships between vendors, products, and booths.
  • Modular architecture: Separation of concerns across database, backend API, and frontend UI layers.
