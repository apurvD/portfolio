Fringe Benefits Application

A full-stack web based application that digitizes the traditional fringe benefits management process for employees and departments.

Overview

The Fringe Benefits Application is a comprehensive full-stack web application designed to streamline the management of employee fringe benefits within an organization. Fringe benefits are forms of compensation provided to employees or related parties in addition to regular wages. These benefits can take many forms—such as event tickets, travel, club memberships, or reimbursements for certain expenses—and are considered part of the overall value an individual receives from the university.
Users can properly report taxable fringe benefits in compliance with IRS regulations. It provides a user-friendly interface for departments to view and manage benefits for employees, while also offering administrative tools for Payroll to oversee and allocate resources effectively.

Problem Addressed

Traditional fringe benefits management often involves manual processes, paper forms, and disparate systems that can lead to compliance risks. This process was carried out on paper forms. By outlining consistent steps and responsibilities, the procedure helps ensure that all benefits provided beyond standard wages are handled accurately and transparently, supporting both institutional accountability and federal compliance.

Key Features

  • Home screen: Departments can submit fringe benefit requests for employees using this page. The form has many dependent dropdowns, and data is stored in JSON.
  • History screen: The Fringe Benefits form History screen shows request id, grantor department, department contact, employee name, benefit type, date submitted, status, and taxability. It also allows the user to filter the list of submissions. The History tab at the top of the screen lets departmental users view all form submissions within their fund-org security access. It is a helpful place to check the status of each form and see the taxability determination for each fringe benefit. .
  • Booth scheduling: Assign and manage booth slots for vendors across different dates or events.
  • Analytics: Aggregate vendor performance metrics such as product counts, inventory status, and activity.
  • Customer-facing views: Browse products and vendors to plan visits and discover merchandise ahead of time.

Technologies & Architecture

Development

  • IDE: Visual Studio Code
  • Source Control: GitLab
  • API Testing: Postman

Database Layer

  • MySQL 8.0: Relational DBMS with ACID compliance using the InnoDB storage engine.
  • Docker Compose: Containerized database deployment for consistent local environments.
  • Key capabilities: Foreign key constraints, transaction support, and normalized schema design.

Backend Layer

  • Node.js & Express: RESTful API server for vendor, product, and analytics endpoints.
  • MySQL2 driver: Database connectivity with parameterized queries for secure data access.
  • API endpoints: Vendor management, product CRUD operations, and analytics aggregation.

Frontend Layer

  • React 18: Component-based UI for dashboards and product views.
  • Tailwind CSS: Utility-first styling for responsive, consistent layouts.
  • React Router: Client-side routing for navigation between vendor dashboards, product pages, and other views.

Implementation Highlights

  • Containerized setup: Application and database run in Docker containers for reproducible environments.
  • Data integrity: Use of foreign keys and transactions to maintain consistent relationships between vendors, products, and booths.
  • Modular architecture: Separation of concerns across database, backend API, and frontend UI layers.
